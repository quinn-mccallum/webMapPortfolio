Lab 1 Write Up

The geojson object represents geospatial vector data for web mapping. All the geographic features (points, lines and polygons) are detailed within the main object of type “featureCollection”. Each feature is then listed as an object with keys “type”, “properties” and “geometry”. “Feature” is the value of each “type” key, denoting a geographic feature belonging to the featureCollection. The properties key holds attribute information (non-geospatial information) about that particular feature, however, this geojson file does not contain property information for its features and is therefore left as an empty object. The geometry key holds an object specifying type and coordinates. The type of feature being specified will be a point, lineString or polygon, etc. Coordinate values are listed under the coordinates key.

A geojson file contains all it’s features in one file regardless of the type of feature, and each feature contains all the necessary property and geospatial data in one place. Alternatively, shapefiles keep different feature types, as well as attribute information, in multiple separate files linked by index files and displayed as layers. The same geospatial data represented in both a geojson file and a shapefile are comparable in file size (with the geojson file being 100mb larger than the shapefile), although the shapefile takes up 4x more disk space than the geojson file.

Geojson files are advantageous to shapefiles because all the data is contained in one, human-readable text file. They are more accessible to developers because geojson files support parsing by most programming languages. Geojson can also feature topology through topojson. However, the single, large file format of geojson may lead to slower load times than shapefiles. In addition, many platforms support shapefiles as this file type has been the benchmark for many years since it was developed 30+ years ago.
